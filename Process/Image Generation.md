After successfully completing the image generation experiments, I began exploring the use of a fixed input image to map into the model, replacing the originally used random seed. Through this approach, I aimed to generate outputs that align more closely with the visual features of my input image.This fixed-input approach aligns more closely with my expectations for a complete image generation workflow. I aim to use this method to create outputs that are meaningfully connected to the input images, allowing the results to reflect both the structure and style of the original input.

I had previously tested this method in the Week 7 notebook. Since my current training model is based on StyleGAN2, I continued using the projection and mapping code provided by NVlabs to carry out these tests.

![qtree-9](https://git.arts.ac.uk/24006172/Critical-Coding-3-Finalproject/blob/main/IMAGE/60.png)

When I initially attempted image projection, the generated outputs appeared very “distorted” . I suspected that the issue might be related to the num_steps parameter, so I increased it from 500 to 5000 and ran the projection again. However, the results remained highly unrealistic. At that point, I began analyzing other possible factors that might be causing the problem.

![qtree-9](https://git.arts.ac.uk/24006172/Critical-Coding-3-Finalproject/blob/main/IMAGE/666.jpg)
![qtree-9](https://git.arts.ac.uk/24006172/Critical-Coding-3-Finalproject/blob/main/IMAGE/61.png)
![qtree-9](https://git.arts.ac.uk/24006172/Critical-Coding-3-Finalproject/blob/main/IMAGE/62.png)

finally I considered that the issue might be caused by a mismatch between the dimensions of my input image and those expected by the pre-trained model. As a result, I adjusted the image size and retrained the model accordingly.

![qtree-9](https://git.arts.ac.uk/24006172/Critical-Coding-3-Finalproject/blob/main/IMAGE/66.png)

After adjusting the input image size, the generated outputs appeared more normal(at least in terms of dimensions and alignment). I recalled that the images used in the Week 7 projection experiments had the same resolution, which I had previously overlooked.

Once the size issue was resolved, I began increasing the num_steps parameter in an attempt to improve the quality of the projected images. I raised it from 500 to 5000, and eventually up to 50,000. However, I noticed that even with such a significant increase in steps, the image quality did not improve substantially. This led me to infer that, at the current training stage, the projection model’s ability is still limited and not yet capable of perfectly reconstructing the input image.
